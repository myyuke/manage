人员管理API

修订

注意点：

本API的职业表内容需要自行添加，以管理员字段为管理员权限的代表。
且进行操作时，职业字段和ID不允许不对应，会提示错误。
初始化初始操作用户的token为888888，用于初始化进行增删改查。

v1.4

新增缓存机制

概述

—–

域名

http://127.0.0.1:8081

数据格式

所有发送和接收的数据都是JSON格式.

HTTP方法

本API采用了REST风格，因此对于资源增删改查是利用HTTP原生的方法进行操作的.

方法 描述

GET

获取资源

POST

创建资源

PUT

更新资源

DELETE

删除资源

管理员后台登入

URI

/login

方法

Post

URI参数

无

BODY参数

参数名称 类型 必填 描述

username

int

是

用户名

password

string

是

密码

示例

{
    "username":"mkmk",
    "password":"123456"
}

响应

{
  "personId": 1,
  "personName": "小草",
  "personAge": 15,
  "position": [
    {
      "id": 3,
      "positionName": "管理员"
    },
    {
      "id": 1,
      "positionName": "学生"
    }
  ],
  "token": "02ca9e36-aa0a-464b-98f0-a904efe3a90c",
  "username": null,
  "password": null
}

账号管理

人员添加

URI

/persons

方法

POST

URI参数

无

BODY参数

参数名称 类型 必填 描述

personName

string

是

人员名称

personAge

int

是

人员年龄

position

array object

是

人员职位（可多个）

token

string

是

接入凭证

username

string

是

账号

password

string

是

密码

示例：

{
    "personName": "小草",
    "personAge": 15,
    "username":"mkmk",
    "password":"123456",
    "position": [
      {
          "id":1,
          "positionName":"学生"
      },
      {
          "id":3,
          "positionName":"管理员"

      }
    ],
    "token":"f6ede775-5db3-44f5-b6b5-ef2f59b18b9f"
}

响应

{
  "code": 200,
  "message": "添加成功",
  "data": {
    "personId": 25,
    "personName": "小草",
    "personAge": 15,
    "position": [
      {
        "id": 1,
        "positionName": "学生"
      },
      {
        "id": 3,
        "positionName": "管理员"
      }
    ],
    "token": "",
    "username": "mkmk",
    "password": "123456"
  }
}

人员删除

URI

/persons

方法

DELETE

URI参数

参数名称 类型 必填 描述

id

int

是

人员id

BODY参数

无

示例：

http://127.0.0.1:8081/persons?id=25&&access_token=f6ede775-5db3-44f5-b6b5-ef2f59b18b9f

响应

{
  "code": 200,
  "message": "删除成功",
  "data": null
}
